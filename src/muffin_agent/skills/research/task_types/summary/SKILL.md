# Task Type — Summary

The writer will produce a **tight, condensed answer** capped at
roughly 500 words.  Your job: gather the **3-5 highest-signal facts**.
Don't drown the writer in evidence; less is more.

## What to gather

- **The headline finding** (one chunk) — the single most important
  thing the user should know.
- **2-4 supporting facts** — the next-most-load-bearing claims.
- **Optional: one "context" chunk** — a quick framing of who/what/why,
  if the user might not already know.

That's it.  4-5 chunks total.  Don't go deeper.

## Source priorities

1. **Authoritative primary source** for the headline finding.
2. **Secondary corroboration** if the headline is contested or
   recent.

## Common pitfalls

- Treating a summary like a research-report.  The writer is going
  to cut everything you over-supply — you're wasting your budget
  and the writer's prompt space.
- Burying the headline.  The user wants the punchline first; make
  sure the most important chunk has the most informative ``snippet``.
- Citing controversial sources for non-controversial facts.  Pick
  the one most-authoritative source per claim.
