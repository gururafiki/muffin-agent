# Task Type — Factual Q&A

The writer will produce a **direct, short answer** — the punchline
in the first sentence, then 1-3 short paragraphs of context.  Your
job: get the single most-authoritative source for the answer, plus
one cross-check.

## What to gather

- **The fact itself** — one chunk from a highly authoritative source
  (official page, Wikipedia for uncontroversial facts, original
  filing).
- **A cross-check** — one independent source confirming the same
  fact.  Skip if the primary source is the canonical authority
  (e.g. a company's official page about itself).

Two chunks is usually plenty.  Three is the upper bound — more is
waste.

## Source priorities

1. **Canonical primary** — for "who is the CEO of X?" that's the
   company's About page.  For "when was Y released?" it's the
   product / album / book's official launch announcement.
2. **Authoritative aggregators** — Wikipedia, official registries,
   government databases.
3. **News reports** — only for recent events not yet captured by
   aggregators.

## Common pitfalls

- Pulling 10 sources for a question that has one canonical answer
  — wasteful and dilutes the writer's citation list.
- Citing a low-quality secondary source when a primary exists.
- Conflating two distinct facts (e.g. "founded by" vs "co-founded
  by"); read carefully and match the exact wording of the user's
  query.
- Stale data — for time-sensitive facts (current price, current
  CEO), ensure the source is recent.
